# 终端透传到公网

将本地终端安全地暴露到公网，实现远程协助、服务器管理、开发调试等场景。

**适用场景：** 统一管理、适配 Claude Code 和 OpenCode 等 AI 编程工具。

## 一、使用效果

### 1. 快速启动

```bash
# 下载客户端（以 Linux amd64 为例）
wget https://github.com/friddle/gotty-piko/releases/latest/download/gottyp-linux-amd64 -O ./gottyp
chmod +x ./gottyp

# 一键启动
./gottyp
```

### 2. 启动后输出示例

```
========================================
Remote URL: https://clauded.friddle.me/friddle-project-a1b2/
Username:   x7kq3m
Password:   p9w2nfc8h4
Port Proxy: https://clauded.friddle.me/friddle-project-a1b2/port/3000
Files:      https://clauded.friddle.me/friddle-project-a1b2/files/
========================================
```

### 3. 浏览器访问效果

打开 `Remote URL`，输入用户名密码后：

- **Web 终端**：直接在浏览器中使用 Shell，支持 zsh/bash/sh
- **文件浏览**：访问 `/files/` 路径，可下载/查看本地文件
- **端口代理**：访问 `/port/{port}`，将本地服务暴露到公网
- **桌面通知**：本地 `notify-send` 会推送到浏览器右下角（适合长时间运行的任务）
- **持久会话**：基于 tmux，刷新浏览器或断网后重新连接，会话不丢失

### 4. 与 AI 工具配合

Claude Code / OpenCode 等工具可以直接访问该 URL：

```bash
# 在 AI 工具中告知访问地址
请访问 https://clauded.friddle.me/friddle-project-a1b2/
用户名: x7kq3m
密码: p9w2nfc8h4
```

AI 可以：
- 在浏览器终端中执行命令
- 通过文件浏览查看代码/日志
- 通过端口代理访问本地的 Web 服务（如 3000 端口的开发服务器）

## 二、自建服务器（Self-Host）

**强烈建议自建服务器！** 公共服务器仅供体验，生产环境请自行部署。

### 1. Docker Compose 部署

```yaml
# docker-compose.yaml
version: "3.8"
services:
  clauded:
    image: ghcr.io/friddle/gottyp-piko-server:latest
    container_name: clauded
    ports:
      - "80:80"
    restart: unless-stopped
```

启动：
```bash
docker-compose up -d
```

### 2. 服务端环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `LISTEN_PORT` | `80` | HTTP 服务端口 |
| `PIKO_UPSTREAM_PORT` | `8022` | Piko 上游端口（内部使用） |
| `ENABLE_TLS` | `false` | 是否启用 HTTPS |
| `TLS_CERT_FILE` | - | TLS 证书路径 |
| `TLS_KEY_FILE` | - | TLS 私钥路径 |

### 3. 客户端连接自建服务器

```bash
# 指定自定义服务器地址
./gottyp --remote=https://your-server.com

# 或指定端口
./gottyp --remote=your-server.com:8088
```

## 三、常用参数说明

### 基础参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--session` | 自定义会话ID，默认是 `{用户}_{目录}_{随机数}` | `--session myproject` |
| `--remote` | 服务器地址，默认使用公共服务器 | `--remote=https://your-server.com` |
| `--auth` | 是否开启 Basic Auth，默认开启 | `--auth=false` 关闭认证 |
| `--auth-name` | 自定义用户名 | `--auth-name admin` |
| `--pass` | 自定义密码 | `--pass mypassword` |

### 功能增强参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--static-index` | 开启文件浏览，指定本地目录映射到 `/files/` | `--static-index=/home/user/code` |
| `--attach-port` | 端口代理，将本地端口映射到 `/port/{port}` | `--attach-port=3000` |
| `--terminal` | 指定终端类型 | `--terminal=zsh` |
| `--tmux` | 使用 tmux 保持会话，默认开启 | `--tmux=false` 关闭 |
| `--tmux-session` | 附加到指定 tmux 会话 | `--tmux-session=my-session` |

### 运行模式参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--daemon` | 守护进程模式（后台运行），默认开启 | `--daemon=false` 前台运行 |
| `--pid-file` | PID 文件路径 | `--pid-file=/tmp/gottyp.pid` |
| `--auto-exit` | 24小时后自动退出，默认开启 | `--auto-exit=false` |

### 通知参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--enable-notify` | 拦截 notify-send 推送到浏览器 | `--enable-notify=true` |
| `--notify-webhook` | 飞书/钉钉等 Webhook 地址 | `--notify-webhook=https://open.feishu.cn/...` |

### 安全参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--upstream-key` | Piko 服务端 API Key，用于认证 | `--upstream-key=your-token` |

### 环境变量对应

所有参数都支持通过环境变量设置（大写下划线格式）：

```bash
export SESSION=myproject
export REMOTE=https://your-server.com
export AUTH_NAME=admin
export PASS=mypassword
export STATIC_INDEX=/home/user/code
export ATTACH_PORT=3000
export TMUX=true
export DAEMON=true
export ENABLE_NOTIFY=true
export NOTIFY_WEBHOOK=https://open.feishu.cn/...
```

## 四、使用示例

### 示例 1：开发调试（配合 AI 工具）

```bash
# 启动终端 + 文件浏览 + 端口代理
gottyp --session myapp --static-index=/home/user/myapp --attach-port=3000
```

效果：
- 终端：`https://server/myapp/`
- 代码文件：`https://server/myapp/files/`
- 开发服务器：`https://server/myapp/port/3000`

### 示例 2：远程服务器管理

```bash
# 指定会话名，方便记忆
gottyp --session prod-server --terminal=bash --tmux-session=production
```

### 示例 3：关闭认证（内网环境）

```bash
# 如果在内网或信任环境，可以关闭 Basic Auth
gottyp --auth=false
```

### 示例 4：前台运行（调试）

```bash
# 前台运行，查看日志输出
gottyp --daemon=false
```

### 示例 5：连接到自建服务器

```bash
# 使用自建服务器
gottyp --remote=https://tunnel.mycompany.com --upstream-key=secret-token
```

## 五、会话管理

```bash
# 列出所有 tmux 会话
gottyp tmux list

# 终止所有会话和守护进程
gottyp tmux kill-all
```

## 六、注意事项

1. **会话命名冲突**：公共服务器上 `{用户}_{目录}_{随机数}` 可能重复，建议自建服务器或使用 `--session` 指定唯一名称
2. **安全性**：生产环境务必开启 `--auth` 并设置强密码，或使用 `--upstream-key` 进行服务端认证
3. **超时**：默认 24 小时自动退出，长时间使用请设置 `--auto-exit=false`
4. **tmux**：建议安装 tmux 以获得最佳体验，会话不会因浏览器关闭而丢失
